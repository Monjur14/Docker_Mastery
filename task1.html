Task 01  ·  Beginner

Hello World
Container

Run the hello-world image from Docker Hub to verify Docker is installed and working correctly. Your first container in under 60 seconds.

~5 minutes Docker installed Terminal / Shell
📖 Required concepts
🐳
What is Docker?
Docker is a platform that packages software into containers — lightweight, self-contained units that include everything needed to run: code, runtime, libraries, and config. Unlike a virtual machine, a container shares the host OS kernel, making it fast to start and small in size.
📦
Images vs. Containers
An image is a read-only blueprint — like a recipe or a class definition. A container is a running instance of that image — like a cake baked from the recipe, or an object created from a class. You can run many containers from one image simultaneously.
🏪
Docker Hub (Registry)
Docker Hub is a public registry — a giant store of pre-built images. When you run docker run hello-world, Docker first looks for the image locally. If it's not found, it automatically pulls it from Docker Hub.
⚙️
The Docker Daemon & CLI
Docker has two main parts. The daemon (dockerd) is a background service that manages containers. The CLI (docker) is the command you type in your terminal, which talks to the daemon via a socket.
🔄
Container Lifecycle
A container moves through states: Created → Running → Exited. The hello-world container prints its message, then exits immediately — that's normal. Not all containers run forever; some just do a job and stop.
🏗️
Image Layers
Docker images are built in layers — each instruction in a Dockerfile adds a layer. Layers are cached and shared between images, which makes pulls faster (only download what you don't have) and storage efficient.
🔬 Command anatomy
docker run hello-world
docker

The CLI binary — how you talk to the Docker daemon from your terminal.

run

Subcommand that creates a new container from an image and starts it. Equivalent to docker create + docker start in one step.

hello-world

The image name to use. Docker looks for it locally first, then pulls from Docker Hub (docker.io/library/hello-world:latest).

🪜 Step-by-step guide
1
Verify Docker is installed
Before running a container, confirm Docker is installed and the daemon is running. This prints the installed version.
shell 
docker --version
ℹ️
You should see something like Docker version 24.0.5, build ced0996. If you get "command not found", Docker isn't installed — visit docs.docker.com to install it first.
2
Check the daemon is running
The daemon must be active for any Docker command to work. This command pings it.
shell 
docker info
⚠️
Error response from daemon? The Docker daemon isn't running. On Mac/Windows: open Docker Desktop. On Linux: run sudo systemctl start docker
3
Run the hello-world container
This single command does everything: pulls the image if needed, creates a container, runs it, and prints its output.
shell 
docker run hello-world
4
Read the output
Docker prints a message explaining exactly what it just did. Here's what to expect:
expected output
Unable to find image 'hello-world:latest' locally
latest: Pulling from library/hello-world
719385e32844: Pull complete
Digest: sha256:1408fec...
Status: Downloaded newer image for hello-world:latest

Hello from Docker!
This message shows that your installation appears to be working correctly.

To generate this message, Docker took the following steps:
1. The Docker client contacted the Docker daemon.
2. The Docker daemon pulled the "hello-world" image from the Docker Hub.
3. The Docker daemon created a new container from that image which runs the
executable that produces the output you are currently reading.
4. The Docker daemon streamed that output to the Docker client, which sent it
to your terminal.
✅
If you see "Hello from Docker!" — your installation is working correctly. The first 5 lines (blue above) only appear the first time, when Docker downloads the image.
5
Confirm the image was downloaded
List all images stored locally. You should see hello-world in the list.
shell 
docker images
expected output
REPOSITORY TAG IMAGE ID CREATED SIZE
hello-world latest 9c7a54a9a43c 8 months ago 13.3kB
6
View the stopped container
The container exited after printing. Use -a to list all containers, including stopped ones.
shell 
docker ps -a
expected output
CONTAINER ID IMAGE COMMAND CREATED STATUS NAMES
a1b2c3d4e5f6 hello-world "/hello" 10 seconds ago Exited (0) 9 seconds ago laughing_morse
ℹ️
Exited (0) means the container finished successfully — exit code 0 is success in Unix. The funny name (like laughing_morse) is auto-generated by Docker when you don't specify one.
7
Run it again (no download this time)
Run the same command again. Notice the image is already cached — it starts instantly without re-downloading.
shell 
docker run hello-world
💡
This time Docker skips the "Pulling from library/hello-world" lines. Each docker run creates a brand new container — you now have two stopped containers listed in docker ps -a.
8
Clean up (optional)
Remove the stopped containers and the image to keep things tidy.
shell 
# Remove all stopped containers
docker container prune

# Remove the hello-world image
docker rmi hello-world
🔍 What happened under the hood
Step 01
🖥️
CLI → Daemon
Your terminal sent the command to the Docker daemon via a Unix socket.
Step 02
🔍
Local cache check
Daemon checked if hello-world:latest exists locally. It didn't.
Step 03
⬇️
Pull from Hub
Daemon pulled the image layers from Docker Hub and stored them locally.
Step 04
📦
Container created
A writable container layer was created on top of the read-only image.
Step 05
▶️
Process started
The /hello binary inside the container ran and printed to stdout.
Step 06
⏹️
Container exited
The process finished, the container stopped with exit code 0 (success).
🎯 What you learned
  • How Docker pulls images automatically from Docker Hub when not found locally
  • The difference between an image (blueprint) and a container (running instance)
  • That containers can exit immediately after completing their task — that's normal
  • How to list images with docker images and containers with docker ps -a
  • Exit code 0 means success; each docker run creates a new separate container
  • How to clean up containers and images with prune and rmi
