📖 Required concepts
🔌
Port Mapping — -p host:container
Containers run in their own isolated network. To reach a service from your host machine, you must map a host port to the container port. -p 8080:3000 means "traffic arriving at host port 8080 is forwarded to port 3000 inside the container."
🏷️
Image Tags
A tag is a label attached to an image: name:version. When no tag is specified, Docker defaults to :latest. Tags are how you distinguish versions of the same image — my-node-app:1.0.0 vs my-node-app:latest can point to different layers.
🎯
Detached vs. Foreground Mode
By default, docker run attaches to the container's output — your terminal is "locked" to it. The -d flag runs in detached (background) mode, printing only the container ID and returning your prompt. This is what you want for long-running servers.
📛
Container Names
Docker auto-generates quirky names like laughing_morse unless you provide one with --name. A predictable name makes it easy to reference the container in later commands — docker logs my-app instead of docker logs a1b2c3d4.
🔄
--rm — Auto-remove on Exit
The --rm flag tells Docker to delete the container automatically when it stops. Great for development and one-off runs — keeps docker ps -a from filling up with dead containers. Do not use in production if you need to inspect logs after a crash.
🌡️
Environment Variables at Runtime
You can inject configuration into a running container with -e KEY=VALUE. In Task 14's server, process.env.PORT reads the port from the environment — letting you override it without rebuilding the image.
🔬 Command anatomy
docker run -d -p 8080:3000 --name my-app my-node-app
docker

The Docker CLI — your interface to the daemon.

run

Creates and starts a new container from the specified image in one step.

-d

Detached mode — runs the container in the background and returns your terminal prompt immediately.

-p

8080:3000 — maps host port 8080 to container port 3000. Visit localhost:8080 in your browser to reach the app.

--name

Assigns a human-readable name to the container instead of a random one. Use this name in all subsequent commands.

my-node-app

The image to run — the one you built in Task 14. Docker appends :latest if no tag is specified.

Understanding the -p flag syntax:

Host side (left)
-p 8080:3000

The port on your machine. This is what you type in the browser: localhost:8080. Choose any free port.

Container side (right)
-p 8080:3000

The port the app listens on inside the container. Must match the EXPOSE value in the Dockerfile.

🪜 Step-by-step guide
1
Confirm the image exists
Make sure your Task 14 image is still on disk before trying to run it.
shell 
docker images my-node-app
expected output
REPOSITORY TAG IMAGE ID CREATED SIZE
my-node-app latest a3f8b2c91d02 2 hours ago 178MB
⚠️
Nothing listed? Go back to Task 14 and run docker build -t my-node-app . from inside the my-node-app/ project folder first.
2
Add a version tag
Apply a specific version tag to the image alongside the default :latest. docker tag creates an alias — both names point to the same image layers, no data is duplicated.
shell 
docker tag my-node-app my-node-app:1.0.0
shell 
docker images my-node-app
expected output — two tags, same image ID
REPOSITORY TAG IMAGE ID CREATED SIZE
my-node-app 1.0.0 a3f8b2c91d02 2 hours ago 178MB
my-node-app latest a3f8b2c91d02 2 hours ago 178MB
ℹ️
Notice both rows share the same IMAGE ID. A tag is just a pointer — the underlying layer data is stored once.
3
Run the container in detached mode
Start the container in the background with port mapping and a named handle. This is the core command of the entire task.
shell 
docker run -d -p 8080:3000 --name my-app my-node-app
expected output
3e4f7a9b2c1d8e5f6a0b4c7d9e2f1a3b8c5d7e0f4a2b6c9d1e3f7a0b5c8d
✅
A long hex string means it worked — that's the full container ID. Your terminal prompt returns immediately because of -d.
4
Verify the container is running
Confirm the container started successfully and the port mapping is active.
shell 
docker ps
expected output
CONTAINER ID IMAGE COMMAND STATUS PORTS NAMES
3e4f7a9b2c1d my-node-app "node server.js" Up 5 seconds 0.0.0.0:8080->3000/tcp my-app
ℹ️
The PORTS column confirms the mapping: 0.0.0.0:8080->3000/tcp means all host interfaces on port 8080 forward to the container's port 3000.
5
Hit the app in your browser
Open your browser and navigate to the mapped host port. You should see the response from your Node.js server.
url 
http://localhost:8080
expected browser response
Hello from inside Docker! 🐳
✅
You can also verify from the terminal with curl http://localhost:8080 — you'll see the same response.
6
Inspect the container's logs
The startup log from Node.js is captured by Docker. Fetch it by name — this is why naming your container matters.
shell 
docker logs my-app
expected output
Server running on port 3000
7
Override the port via environment variable
Your app reads process.env.PORT. Pass a different value at runtime with -e to change the port without touching the image. First, stop and remove the existing container.
shell 
# Stop and remove the running container
docker rm -f my-app

# Run again, overriding the port to 4000 inside the container
docker run -d -p 8080:4000 -e PORT=4000 --name my-app my-node-app
shell 
docker logs my-app
expected output — different port, same image
Server running on port 4000
💡
Same image, different runtime config — no rebuild required. This is the value of reading config from environment variables instead of hardcoding it.
8
Inspect the running container's full config
Use docker inspect to see every detail about the running container — its IP address, mounts, environment variables, and port bindings — as structured JSON.
shell 
# Full JSON dump
docker inspect my-app

# Extract just the port bindings with --format
docker inspect --format '{{json .NetworkSettings.Ports}}' my-app
port bindings output
{"4000/tcp":[{"HostIp":"0.0.0.0","HostPort":"8080"}]}
9
Stop and clean up
Stop the running container and remove both the container and the tagged image when you're done.
shell 
# Stop the container gracefully
docker stop my-app

# Remove the stopped container
docker rm my-app

# Optional: remove the image when done with this task
docker rmi my-node-app my-node-app:1.0.0
ℹ️
docker stop sends SIGTERM to the process, giving it a chance to clean up gracefully (10s timeout). docker kill sends SIGKILL immediately — use that only when stop hangs.
📋 docker run flag reference
Flag What it does Example
-d Detached — run in background, return prompt docker run -d …
-p host:container Publish a port from container to host -p 8080:3000
--name Assign a predictable name to the container --name my-app
-e KEY=VALUE Set an environment variable inside the container -e PORT=4000
--rm Auto-delete container when it exits docker run --rm …
-it Interactive + TTY — for shell access docker run -it … sh
--restart Restart policy on crash or reboot --restart unless-stopped
