Task 10  ·  Beginner

Real-time
Resource Stats Monitoring

Monitor CPU, memory, and network usage of active containers live with docker stats. No extra tools, no dashboards — just a live terminal readout straight from the daemon.

~7 minutes Docker installed Terminal / Shell
📖 Required concepts
📊
What docker stats shows
docker stats streams a live, continuously updating table of resource usage for running containers — CPU %, memory usage/limit, network I/O, block I/O, and process count. It's Docker's built-in equivalent of top, but scoped to containers instead of host processes.
🧮
Where the numbers come from
These stats aren't estimated — they're read straight from the Linux kernel's cgroups (control groups), the same mechanism Docker uses to enforce resource limits. Each container's cgroup tracks exactly how much CPU time, memory, and I/O it has consumed.
🔄
Live vs. one-shot
By default docker stats refreshes continuously, similar to watch. Pass --no-stream to capture a single snapshot instead — useful for scripting or logging usage at a point in time.
🎯
Scoping to specific containers
Running docker stats with no arguments shows every running container. Pass one or more container names or IDs to narrow the view to just the ones you care about.
🚦
CPU % can exceed 100%
CPU percentage is relative to one core. A container using two full cores on a multi-core host will report ~200%. This is expected and not a bug — check your host's core count when interpreting the number.
🛠️
Formatting output
The --format flag accepts a Go template, letting you pick exactly which columns to print — handy when piping stats into a script or log file instead of eyeballing the full table.
🔬 Command anatomy
docker stats --no-stream web db
docker

The CLI binary — how you talk to the Docker daemon from your terminal.

stats

Subcommand that streams a live table of resource usage for running containers.

--no-stream

Optional flag — prints a single snapshot and exits, instead of refreshing forever.

web db

Optional list of container names/IDs to scope the output. Omit to see all running containers.

🪜 Step-by-step guide
1
Start a couple of containers to monitor
Run two different containers in detached mode so there's something to compare in the stats table.
shell 
docker run -d --name web nginx
docker run -d --name stress polinux/stress stress --cpu 2 --timeout 300s
ℹ️
The stress container deliberately burns CPU for 5 minutes so you have something visibly active to watch in the stats output.
2
Launch the live stats stream
Run docker stats with no arguments to see every running container update live.
shell 
docker stats
3
Read the live table
Watch how stress pins its CPU % much higher than web, which is idle. The terminal refreshes in place roughly every second.
live output
CONTAINER ID NAME CPU % MEM USAGE / LIMIT MEM % NET I/O BLOCK I/O PIDS
a1b2c3d4e5f6 web 0.02% 3.5MiB / 7.6GiB 0.05% 1.2kB / 0B 0B / 0B 3
f6e5d4c3b2a1 stress 198.7% 12.4MiB / 7.6GiB 0.16% 0B / 0B 0B / 0B 4
💡
Notice stress sits near 200% because --cpu 2 spins up two worker threads. On a multi-core host that's two full cores' worth of usage.
4
Exit the live stream
Press Ctrl+C to stop watching — this only kills the stats stream, not the containers themselves.
ℹ️
docker stats is a read-only view. Closing it never stops, pauses, or affects the containers you're watching.
5
Take a single snapshot instead
Add --no-stream to print one reading and return to your prompt immediately.
shell 
docker stats --no-stream
6
Scope to a single container
Pass a container name to only monitor that one — useful when you have many containers running and only care about one.
shell 
docker stats --no-stream stress
7
Customize the columns with --format
Print just the fields you want using a Go template — handy for piping into a log file or another script.
shell 
docker stats --no-stream --format "table {{.Name}}\t{{.CPUPerc}}\t{{.MemUsage}}"
expected output
NAME CPU % MEM USAGE / LIMIT
web 0.02% 3.5MiB / 7.6GiB
stress 197.4% 12.4MiB / 7.6GiB
8
Clean up
Stop and remove both containers now that you're done monitoring them.
shell 
# Stop both containers
docker stop web stress

# Remove both containers
docker rm web stress
📐 Column reference
Column Meaning
CPU % Percentage of one CPU core currently in use. Can exceed 100% on multi-core hosts.
MEM USAGE / LIMIT Current memory consumption vs. the container's memory limit (host total if none is set).
MEM % Memory usage as a percentage of the limit shown in the previous column.
NET I/O Total bytes received / transmitted over the container's network interface since it started.
BLOCK I/O Total bytes read from / written to block devices (disk) since the container started.
PIDS Number of processes or threads currently running inside the container.
🎯 What you learned
  • How to stream live CPU, memory, network, and disk stats for all running containers
  • That the numbers come directly from Linux cgroups, not estimates
  • How to capture a one-time snapshot with --no-stream
  • How to scope docker stats to specific containers by name
  • Why CPU % can exceed 100% and how to interpret it relative to core count
  • How to customize output columns with --format for scripting
